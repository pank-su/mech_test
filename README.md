# Тренажер по механике

Статический сайт для подготовки к тестам по механике. Вопросы перенесены из PDF в редактируемую структуру данных, схемы и формулы вынесены в отдельные изображения, а оригинальные страницы PDF доступны внутри каждого вопроса для сверки.

## Сайт

https://pank-su.github.io/mech_test/

## Возможности

- 70 вопросов из исходных PDF и дополнительного банка.
- Выбор отдельного теста или прохождение всех вопросов сразу.
- Случайный порядок вопросов по умолчанию, с возможностью переключиться на порядок из банка.
- Поддержка вопросов с одним и несколькими правильными ответами.
- Проверка ответа сразу на странице.
- Сохранение прогресса в `localStorage` браузера.
- Пометки `Решил` и `Не решил` для повторения.
- Выделенные схемы/формулы в карточке вопроса.
- Оригинальная PDF-страница в раскрывающемся блоке.

## Структура проекта

```text
.
├── index.html                  # Разметка страницы
├── styles.css                  # Стили интерфейса
├── app.js                      # Логика тренажера
├── data/
│   └── questions.js            # База вопросов и правильных ответов
├── assets/
│   ├── question-media/         # Выделенные схемы и формулы
│   └── pdf-pages/              # Полные страницы исходных PDF
├── тест1.pdf                   # Исходный PDF
└── тест2.pdf                   # Исходный PDF
```

## Локальный запуск

Можно открыть `index.html` напрямую в браузере. Для проверки через локальный сервер:

```bash
python3 -m http.server 8765
```

После запуска открой:

```text
http://127.0.0.1:8765
```

## Как добавить вопрос

Вопросы лежат в `data/questions.js`. Добавь объект в массив `questions` нужного теста:

```js
{
  id: "test1-016",
  source: "свой источник",
  page: 16,
  type: "single",
  text: "Текст вопроса",
  media: "assets/question-media/test1-016.png",
  options: ["Вариант 1", "Вариант 2"],
  correct: [0],
  pageImage: "assets/pdf-pages/test1-page-16.png"
}
```

Для вопроса с несколькими ответами используй:

```js
type: "multiple"
```

## Как указать правильный ответ

Поле `correct` хранит массив индексов правильных вариантов, начиная с нуля.

Один правильный ответ:

```js
correct: [2]
```

Несколько правильных ответов:

```js
correct: [0, 3]
```

Если у вопроса нет отдельной схемы или полной страницы PDF, можно поставить:

```js
media: null,
pageImage: null
```
